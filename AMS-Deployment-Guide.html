3. Server Setup
🔌 Connect to EC2
ssh -i your-key.pem ec2-user@YOUR_EC2_PUBLIC_IP
📦 Install Node.js 20
# Add NodeSource repo and install
curl -fsSL https://rpm.nodesource.com/setup_20.x | sudo bash -
sudo yum install -y nodejs

# Verify
node --version    # should print v20.x.x
npm --version
⚙️ Install PM2
sudo npm install -g pm2
🗄️ Install MySQL 8
# Install MySQL 8 community server
sudo yum install -y https://dev.mysql.com/get/mysql80-community-release-el7-11.noarch.rpm
sudo yum install -y mysql-community-server

# Start and enable on boot
sudo systemctl start mysqld
sudo systemctl enable mysqld

# Get the temporary root password
sudo grep 'temporary password' /var/log/mysqld.log
🔒 Secure MySQL
sudo mysql_secure_installation
# Follow prompts:
# - Set a strong root password
# - Remove anonymous users: Yes
# - Disallow root login remotely: Yes
# - Remove test database: Yes
# - Reload privilege tables: Yes

5. Database Setup
🗄️ Create the Database
mysql -u root -p
# Enter your MySQL root password
-- Inside MySQL shell:
CREATE DATABASE ams CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
EXIT;
🔄 Push Schema (Prisma db push)
cd /home/ec2-user/myapps/Backend
npm install
npx prisma generate
npx prisma db push
Note: This project uses prisma db push (not migrate deploy). The schema is in Backend/prisma/schema.prisma. Run prisma db push again whenever the schema changes.
🌱 Seed Initial Data
node prisma/seed.js
What seeding creates: System roles (Super Admin) · Permission modules (Users, Roles & Permissions, Settings) · Default admin user account
6. Backend Configuration & Deployment
📝 Create .env File
cd /home/ec2-user/myapps/Backend
nano .env
# Backend Environment Variables
PORT=4000
JWT_SECRET=your-very-long-random-secret-min-32-chars
JWT_EXPIRES_IN=7d
DATABASE_URL="mysql://root:YOUR_MYSQL_PASSWORD@localhost:3306/ams"
ALLOWED_ORIGIN=http://YOUR_EC2_PUBLIC_IP:3000
Important: ALLOWED_ORIGIN must exactly match the URL you open in your browser — including protocol (http://) and port (:3000). Example: http://44.203.116.252:3000
📦 Install Dependencies
npm install
npx prisma generate
🚀 Start with PM2
pm2 start src/server.js --name backend
pm2 save
✅ Verify Backend
curl http://127.0.0.1:4000/
# Expected response: {"message":"AMS API is running"}

8. PM2 Auto-start on Reboot

This ensures PM2 and all processes restart automatically if the server reboots.

# Generate startup script
pm2 startup

# PM2 will print a command — copy and run it. It looks like:
sudo env PATH=$PATH:/usr/bin /usr/lib/node_modules/pm2/bin/pm2 startup systemd -u ec2-user --hp /home/ec2-user

# Save current process list
pm2 save
After this, both backend and frontend will automatically restart after any server reboot — no manual intervention needed.
9. AWS EC2 Security Group

In the AWS Console → EC2 → Security Groups → select your instance's group → Edit Inbound Rules:

TypeProtocolPortSourcePurpose
SSHTCP22Your IP onlyServer access
Custom TCPTCP30000.0.0.0/0Next.js frontend
Custom TCPTCP40000.0.0.0/0Express backend API
Do NOT expose port 3306 (MySQL) to the internet. It must only be accessible from localhost on the server.

12. Deploying Updates
🔄 Backend Update
# Pull latest code
cd /home/ec2-user/myapps
git pull origin main

cd Backend
npm install                          # install any new packages
npx prisma generate                  # always re-generate after pull
npx prisma db push                   # only if schema.prisma changed

pm2 restart backend
pm2 logs backend --lines 20 --nostream   # verify no errors
🔄 Frontend Update
cd /home/ec2-user/myapps/Frontend

npm install                          # install any new packages
npm run build                        # rebuild for production

pm2 restart frontend
pm2 logs frontend --lines 20 --nostream  # verify no errors
Always rebuild the frontend (npm run build) after any code change. Restarting PM2 alone does not apply frontend changes — a new build is required.
13. Useful PM2 Commands
CommandDescription
pm2 statusView all running processes with uptime and CPU/mem
pm2 logs backendTail live logs for the backend
pm2 logs frontendTail live logs for the frontend
pm2 logs backend --lines 50 --nostreamView last 50 lines without tailing
pm2 restart backendRestart the backend process
pm2 restart frontendRestart the frontend process
pm2 stop backendStop the backend (without removing it)
pm2 delete backendRemove the backend from PM2
pm2 flushClear all log files
pm2 saveSave current process list for reboot persistence
pm2 killStop the PM2 daemon entirely
14. Troubleshooting
ProblemLikely CauseFix
Backend exits immediately Zombie node process holding port 4000 Run ss -tlnp | grep 4000 → kill -9 <pid> → restart PM2
Login hangs / CORS error ALLOWED_ORIGIN mismatch Check .env — ALLOWED_ORIGIN must exactly match the browser URL
Frontend can't reach API Wrong NEXT_PUBLIC_API_URL Check .env.local — must point to EC2 IP port 4000, not localhost
Database connection error MySQL not running or wrong credentials sudo systemctl status mysqld → check DATABASE_URL in .env
npm run build fails Missing env vars or code error Check .env.local exists and NEXT_PUBLIC_API_URL is set
PM2 shows "errored" App crash on startup pm2 logs backend --lines 50 --nostream to see the error
curl hangs on localhost IPv6 vs IPv4 resolution Use curl http://127.0.0.1:4000/ instead of localhost
Port already in use Old process not cleaned up ss -tlnp | grep PORT → kill -9 <pid>
Prisma db push error Wrong DATABASE_URL or MySQL down Verify MySQL is running and DATABASE_URL password is correct. Run npx prisma generate first, then npx prisma db push

16. Deployment Checklist

Use this checklist to confirm every step is complete before handing over to the client.

🖥️ Server
  • EC2 instance is running and accessible via SSH
  • Node.js v20 is installed (node --version)
  • PM2 is installed globally (pm2 --version)
  • MySQL 8 is installed and running (sudo systemctl status mysqld)
  • MySQL root password is set (secure, not default)
  • EC2 Security Group allows ports 22, 3000, 4000
  • Port 3306 (MySQL) is NOT exposed publicly
🗄️ Database
  • ams database created in MySQL
  • Schema pushed to DB (npx prisma generate && npx prisma db push)
  • Seed script run (node prisma/seed.js)
  • Admin user exists in database
⚙️ Backend
  • .env file created with all required variables
  • JWT_SECRET is a strong random string (not a placeholder)
  • ALLOWED_ORIGIN matches the exact frontend browser URL
  • DATABASE_URL points to correct MySQL credentials
  • npm install completed without errors
  • PM2 backend process is online
  • curl http://127.0.0.1:4000/ returns API response
🌐 Frontend
  • .env.local file created with NEXT_PUBLIC_API_URL
  • URL points to EC2 IP (not localhost)
  • npm install completed without errors
  • npm run build completed successfully
  • PM2 frontend process is online
  • Login page loads in browser
✅ Final Verification
  • PM2 startup configured (auto-restart on reboot)
  • pm2 save run after startup configuration
  • Login works with admin@admin.com / 123456
  • Default admin password changed
  • Dashboard loads and displays user/role counts correctly
  • New user can be created and logged in successfully
  • Role permissions toggled and verified on next user action
  • Company name updated in Settings and reflected in the header

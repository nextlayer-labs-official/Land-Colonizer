1. Overview

REMS is a web-based Real Estate Management System for tracking property purchases, customers, and sales — built on top of an Access Management System (AMS) that controls user accounts, roles, and permissions across your organisation.

The platform provides role-based access control (RBAC) so each team member sees only the modules and actions their role permits. All modules use an Odoo-style form interface: a record opens in view (read-only) mode, and you click Edit to make changes.

🏠 Purchases

Track every plot, land, or shop purchased — area, rate, payments, brokerage, registration details, and seller information.

👥 Customers

Maintain a customer directory with contact details, ID proof records, and a full sales history per customer.

📋 Sales

Record sales of purchased plots to customers — sale amount, instalments, commissions, expenses, registration, and computed net income.

⚙️ Admin (AMS)

Manage users, roles, permissions, and company settings. Controls who can access each module and what they can do.

2. Navigation

REMS uses a top navigation bar across all pages (dark purple bar at the top). The bar contains:

  • Waffle icon (top-left) — application menu icon
  • Company name — set in Settings, visible to all users
  • Nav links — Dashboard · Projects · Purchases · Customers · Sales · Users · Roles · Settings
  • User avatar (top-right) — click to see your name, email, and Logout button
Permission-filtered nav: Links you don't have VIEW permission for are automatically hidden. A Manager who only has Purchases access will only see Dashboard and Purchases.

5. How the Form Interface Works

All record modules (Purchases, Customers, Sales) use the same Odoo-style interface. Understanding this pattern makes every module easy to use.

📋 List View

Each module opens to a list view showing all records in a table. From here you can:

  • Click the New button to create a new record
  • Click any row to open that record's detail page
  • Use the Search bar to filter records
  • Use checkboxes to select records for bulk delete
  • Use the ← → pagination arrows at the bottom
👁️ View Mode (Record Detail Page)

When you click a row, the record opens in view (read-only) mode. All fields are displayed but not editable. The top bar shows:

  • Breadcrumb — e.g. Purchases / Plot P-42 — click the module name to go back to the list
  • Status Pipeline — shows the current stage of the record (Draft → In Progress → Registered → Completed)
  • Edit button — click to switch to edit mode (only visible if you have EDIT permission)
  • Action menu — click to access New and Delete options
✏️ Edit Mode

Click Edit to switch the record to edit mode. An amber "Editing" badge appears on the record header. The top bar now shows:

  • Save — saves all changes and returns to view mode
  • Discard — cancels all unsaved changes and returns to view mode

A second Save / Discard bar also appears at the bottom of the form for convenience when scrolled down.

After a successful save, a green "Saved" indicator briefly appears in the top bar.
➕ New Record Page

Clicking New opens a dedicated full-page create form. The top bar shows:

  • Save — creates the record and redirects to its view page
  • Save & New — creates the record and immediately opens a blank form for the next one
  • Discard — discards the unsaved new record and returns to the list
📊 Status Pipeline

The status pipeline in the top bar shows where each record stands. It is automatically calculated — you do not need to set it manually.

✏️ Edit a Purchase
  1. Open the purchase from the list (click its row)
  2. Click the Edit button in the top bar
  3. Update the required fields
  4. Click Save (or Discard to cancel)
🗑️ Delete a Purchase
  1. Open the purchase record (click its row in the list)
  2. Click Action → Delete
  3. Confirm the deletion in the dialog
Warning: Deleting a purchase is permanent. If a Sale is linked to this purchase, delete or unlink the sale first.
🔍 Filter & Search

On the Purchases list, use the toolbar to:

  • Search — filter by plot number, location, seller, or broker name
  • Filters dropdown — filter by Type (Plot / Land / Shop) or Status (Active / Inactive)
  • Pagination — 15 records per page, navigate with ← → arrows

8. Sales

Required permissions: SALE_VIEW · SALE_CREATE · SALE_EDIT · SALE_DELETE

The Sales module records the transaction when a purchased plot is sold to a customer. It links a Customer and a Purchase (plot), then tracks sale amount, instalments, commissions, expenses, registration, and computes net income automatically.

➕ Create a New Sale
  1. Click Sales in the top navigation bar
  2. Click the New button
  3. Select a Customer from the dropdown
  4. Select a Plot / Purchase from the dropdown — the Sale Area and Sale Rate fields will auto-fill from the purchase record
  5. Adjust the sale rate and area if the sale terms differ from the purchase
  6. Fill in payment, commission, expenses, and registration details
  7. Click Save
Auto-fill: When you select a plot, the Sale Area and Sale Rate are automatically populated from the linked Purchase record. You can still override them manually.

9. Roles & Permissions

Every user is assigned exactly one role. The role determines which navigation links are visible and what actions are allowed. Roles are managed by a Super Admin from the Roles page.

Super Admin

Full unrestricted access to all modules. This is a system role — it cannot be edited, deleted, or have permissions restricted.

Custom Roles

Any number of roles can be created (Admin, Manager, Agent, Viewer). Each role has individual permissions toggled per module.

Admin role note: The built-in Admin role has is_system = false. It relies on its assigned permission codes, not a system bypass. Ensure the correct permissions are toggled on for each role.

All available permission codes:

10. User Management

Required permissions: USER_VIEW · USER_CREATE · USER_EDIT · USER_DELETE

No self-registration: Users cannot sign up on their own. All accounts are created by an Admin. Share login credentials securely with new team members.
➕ Add a New User
  1. Go to Users in the top navigation bar
  2. Click the New button in the control panel
  3. Fill in: Full Name, Email, Phone (optional), Password, Role, Status
  4. Click Save
The user can log in immediately with the credentials you set. Share the password securely and ask them to change it using the Forgot Password flow.
✏️ Edit a User
  1. Find the user in the Users list
  2. Click the row to open their profile, then click Edit
  3. Update any field: Name, Email, Phone, Role, Status, or Password
  4. Click Save to apply changes
To revoke access without deleting the account, set Status to Inactive. Inactive users cannot log in.
🗑️ Delete / Bulk Delete Users
  • Single delete: Click the trash icon on hover, or use the delete option in the user profile
  • Bulk delete: Check multiple rows, then click the red Delete (N) button that appears in the control panel
11. Roles Management

Required permissions: ROLE_VIEW · ROLE_CREATE · ROLE_EDIT · ROLE_DELETE

➕ Create a Role
  1. Go to Roles in the top navigation bar
  2. Click New, enter a role name, and click Save
  3. Click the role row to expand its permissions panel
  4. Toggle on the required permission codes for each module
  5. Changes save immediately — no Save button needed for permission toggles
The Super Admin role is a system role and cannot be edited, deleted, or have permissions restricted.
🗑️ Delete a Role
  1. Click the delete icon on the role row (hover to reveal)
  2. Confirm the deletion
Caution: Deleting a role that is assigned to users may remove their access. Reassign those users to another role before deleting.
12. Company Settings

Available to: Super Admin only

  1. Go to Settings in the top navigation bar
  2. Update the Company Name field
  3. Click Save — the new name appears in the navigation header immediately for all users

15. Frequently Asked Questions
QuestionAnswer
I can't see Purchases / Customers / Sales in the navigation bar. Your role doesn't have the VIEW permission for that module. Ask a Super Admin to open Roles, find your role, and enable PURCHASE_VIEW / CUSTOMER_VIEW / SALE_VIEW.
The Edit button is not visible on a record page. Your role has the VIEW permission but not EDIT. Ask a Super Admin to grant you the EDIT permission (e.g. PURCHASE_EDIT) for the relevant module.
The New button is not visible on a list page. Your role does not have the CREATE permission for that module. Ask a Super Admin to enable PURCHASE_CREATE / CUSTOMER_CREATE / SALE_CREATE.
The Total Amount / Sale Amount fields are blank even after I enter a Rate and Area. These are computed fields — they calculate automatically. Make sure both Rate and Area have numeric values (no text or symbols). The display updates as you type.
I selected a plot in a Sale form but the Area and Rate did not auto-fill. The auto-fill pulls values from the linked Purchase record. Ensure the Purchase has an Area (Purchased Area) and Rate saved. If the purchase was created without them, edit the purchase first, then return to the sale form.
The status pipeline still shows "Draft" after I entered an Advance. The pipeline is computed from the saved data. Click Save first — the pipeline updates after the data is persisted.
How do I reach "Completed" status? A record reaches Completed when 100% of the amount is paid/received AND a Registration Date is entered and saved.
I opened a record but all the fields are grayed out and I can't edit anything. Records open in view (read-only) mode by default. Click the Edit button in the top bar to switch to edit mode. If the Edit button is missing, your role does not have EDIT permission.
I saved a record but then clicked Discard — did my changes get lost? Discard only cancels unsaved edits in the current edit session. If you already clicked Save, those changes are persisted. Discard after a Save will just reset the form to the saved (already-updated) values.
Can I link one plot to multiple sales? Yes. The system allows multiple Sale records linked to the same Purchase. This covers scenarios like partial plot sales or re-sales.
The Sales History panel is not showing on a customer's page. The panel only appears if at least one sale is linked to that customer. Create a Sale and select this customer — it will then appear in the history.
I can't see a navigation link (Users, Roles, Settings). Your role doesn't have the VIEW permission for that module. Ask a Super Admin to update your role's permissions under Roles.
I forgot my password and can't log in. Click Forgot password? on the login page, enter your email, and follow the reset link in the email. No Admin involvement needed.
How do I reset another user's password? Open the user via the Users list, click Edit, set a new password in the Password field, and click Save. No email is sent — share the new password directly.
How do I prevent a user from logging in without deleting them? Edit the user and set their Status to Inactive. Their account and history are preserved but they cannot log in.
Can I edit the Super Admin role? No. The Super Admin role is a permanent system role with full access. It cannot be modified or deleted.
After restarting the backend, some nav items disappeared. If the backend was down when the page loaded, permission fetching may have failed. Log out and log back in — this triggers a fresh permission fetch from the server.
